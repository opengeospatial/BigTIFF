<title id="_1de7d78b-227f-fc20-e618-fca8c6c3421a">BigTIFF requirements</title>
<p id="_36e9c606-6c49-062b-4d95-43a7335c8072">The BigTIFF standard provides a solution for a TIFF file that requires a file size larger than 4 Gigabytes. In principle, the BigTIFF standard will also support smaller files. However, files that do not require file size bigger than 4 Gigabytes or not using the new data types, should use TIFF version 6 specification instead. This is in order to ensure backwards compatibility with old TIFF implementations not supporting BigTIFF.</p>

<p id="_0e1249b3-ee75-9a38-8303-1a3d2be5a7ec">A BigTIFF file (and a TIFF) is defined as a sequence of 8-bit bytes, where the bytes are numbered from 0 to N. In a BigTIFF file, the largest possible TIFF file is 2<sup>64</sup> bytes in length.</p>
